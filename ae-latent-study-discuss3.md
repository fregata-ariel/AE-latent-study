## まず大事な点

2. 普通の point-cloud PH は、

   * orbifold point
   * cusp
   * stabilizer の情報
     をほぼ無視します。
     つまり「モジュライ的に重要な構造」と「Euclidean point cloud として見える構造」はずれます。

3. 有限サンプルと密度の偏りだけで、ループっぽいものは簡単に出たり消えたりします。

なので、このサイド実験の狙いは

## さらに重要な注意

latent の「第1軸、第2軸…」には通常、固有の意味がありません。
回転しても同じ表現だからです。

なので、単純に

* z7 を消す
* z6 を消す

のようなやり方は、あまり解釈できません。

「1つずつ落とす」をやるなら、まず latent に**順序づけされた基底**を入れる必要があります。
おすすめは次のどれかです。

* **PCA 基底**
  まず一番素直です。分散の大きい方向から残す。
* **局所幾何ベースの基底**
  近傍保存を最大化する方向を残す。
* **観測量ベースの基底**
  `log|j|` や orbit 識別に寄与する方向を優先する。

最初は PCA で十分です。
まずは「高次元 latent を PCA で 8→7→6→…→1 に切る」が一番実装しやすいです。

* phase jitter
* timing jitter
* amplitude modulation noise

# 提案: `PhaseTorus12`

## 1. 基本アイデア

観測は 1 次元時系列です。
コアは **2つの独立位相**で作る `T^2` 構造にします。
その上に

* 振幅・包絡・ドリフト・歪み
* 低周波 / 帯域制限 / 高周波ノイズ

を重ねて、全部で 12 因子にします。

これなら

* clean 条件では「ほぼ torus」
* ノイズを入れると PH がどう壊れるか
* latent 次元を落とすと、どの因子から消えるか

が読みやすいです。

---

## 2. 生成式

時刻 `t_n = n/N`, `n=0,...,N-1`、たとえば `N=512` とします。
基底周波数は固定で `f1=5`, `f2=11`, `f_env=1` くらいでよいです。

信号本体を

[
s(t)
====

\lambda,
\Bigl(1 + e \cos(2\pi f_{\mathrm{env}} t + \phi)\Bigr)
\Bigl(
\cos(2\pi f_1 t + \theta_1)
+
r \cos(2\pi f_2 t + \theta_2)
+
\kappa \cos(2\pi (2f_1)t + 2\theta_1)
\Bigr)
+
b,(t-\tfrac12)
]

とします。

観測は

[
x(t)= s(t) + n_{\mathrm{low}}(t) + n_{\mathrm{band}}(t) + n_{\mathrm{high}}(t)
]

です。

---

## 3. 12 因子

### コア因子

1. `θ1 ∈ S^1`
2. `θ2 ∈ S^1`

ここが主役です。
clean 条件では真のコア位相構造は `T^2` です。

### 幾何因子

3. `λ` : 全体スケール
4. `r` : 第2成分の相対振幅
5. `e` : 包絡の深さ
6. `φ` : 包絡位相
7. `b` : 線形ドリフト強度
8. `κ` : 2次高調波歪みの強さ

### ノイズ因子

9. `σ_low` : 低周波ノイズ強度
10. `σ_band` : 帯域制限ノイズ強度
11. `ν_band` : 帯域制限ノイズの中心周波数
12. `σ_high` : 高周波ノイズ強度

これで 12 因子です。

---

# 4. 因子の役割

## コア

`θ1, θ2` は **位相的主構造**です。
ここが残っている限り、latent は本質的には torus 的な情報を持っています。

## 幾何

`λ, r, e, φ, b, κ` は、位相そのものではないが観測形状をかなり変える因子です。
latent がうまく学べていれば、これらはコア位相の外側に「厚み」として乗るはずです。

## ノイズ

`σ_low, σ_band, ν_band, σ_high` は nuisance です。
理想的には最初に捨てられてほしい自由度です。
ここが最後まで残るようだと、モデルが本質より毛羽立ちを拾っていることになります。

---

# 5. 群・幾何の見取り図

clean 側の本質は

[
T^2 \times \mathbb{R}_+ \times I^4 \times \mathbb{R}
]

のような感じです。
厳密な意味で全因子が群とは限りませんが、少なくともコアは

[
G_{\mathrm{core}} = S^1 \times S^1
]

で、**低次元因子に分解された群**を明示的に持っています。

このデータセットの良さは、
「まず `T^2` がある」と分かっていることです。
PH の議論がぶれにくいです。

---

# 6. clean 条件で期待する PH

理想的には `T^2` なので、

* `H0`: 1 成分
* `H1`: 2 本の主要ループ
* `H2`: 1 つの面成分

が期待されます。

ただし実務上は、`H2` はかなり不安定です。
なので最初の主指標は

* **長寿命の H1 が 2 本見えるか**
* `k` 次元に落としたときに、その 2 本のうち何本が残るか

でよいと思います。

たとえば次のくらいで十分です。

[
\theta_1,\theta_2,\phi \sim \mathrm{Unif}(0,2\pi)

[
\lambda \sim \mathrm{Unif}(0.8,1.2),\quad
r \sim \mathrm{Unif}(0.5,1.5)

[

[

`ν_band` は Nyquist の 10%〜40% くらいに置けば十分です。

# 8. ノイズの具体化

## 低周波ノイズ

ゆっくりしたランダム和音か、低次 Fourier 成分だけを残したガウス過程で作る。

[
n_{\mathrm{low}}(t)
===================

\sigma_{\mathrm{low}}
\sum_{m=1}^{M_{\mathrm{low}}}
a_m \cos(2\pi m t + \psi_m)
]

`M_low` は 2〜4 くらい。

## 帯域制限ノイズ

中心周波数 `ν_band` と帯域幅 `w_band` の近傍だけ通す。

[
n_{\mathrm{band}}(t)
====================

\sigma_{\mathrm{band}} \cdot \text{bandpass-noise}(\nu_{\mathrm{band}}, w_{\mathrm{band}})
]

## 高周波ノイズ

white noise を高域通過して作るか、単純に白色ノイズでもよいです。

[
n_{\mathrm{high}}(t)=\sigma_{\mathrm{high}} ,\varepsilon(t)
]

# 9. 期待される退化順序

ここが本題です。

## 理想的な退化順序

高次元 latent から 1 つずつ落としていくとき、理想的には次の順で情報が落ちます。

### 第1段階

まずノイズ因子が落ちる
`σ_high`, `σ_band`, `ν_band`, `σ_low`

ここでは PH の長寿命バーはほぼ維持され、
短寿命バーだけ減るのが理想です。

### 第2段階

次に冗長な幾何因子が落ちる
`λ`, `b`, `κ`

これは局所厚みや散らばり方に効きますが、
コア位相はまだ残るはずです。

### 第3段階

次に包絡因子が縮退する
`e`, `φ`

この段階で point cloud の厚みがかなり薄くなります。
ただし、うまくいっていれば `T^2` の骨格は残ります。

### 第4段階

最後にコア位相が壊れ始める
`θ1`, `θ2`

ここで初めて、長寿命 H1 が 2 本から 1 本、さらに 0 本へ落ちるはずです。

を `m=1,2,...` で出す。

するとたとえば `lattice_standard_norm_inv` に対して、

* `m=1` でも orbit はかなり保つが chart は薄い
* `m=2` で chart が少し改善
* `m=3` 以上は主に冗長度

のような読みができるようになります。

# かなり重要な観点

今の `eff_dim` は有用ですが、
**“2に近いほど良い”** を常に一般化すると危険です。

Lattice では「2D 商空間らしさ」を見たいから 2 が基準ですが、
合成データ `PhaseTorus12` では

* コア位相は 2
* 厚みや nuisance を含めると実効次元はもっと大きい

かもしれません。

なので統一基盤では `eff_dim` は

* 良し悪しを直接言う主指標ではなく
* **表現の厚みを見る診断量**

に下げた方が安全です。

主指標はむしろ

* orbit
* obs
* topo
* それらの最小十分次元

に置く方がよいです。

# 統一的な順位づけ

これも今後かなり効きます。
私は単一スコア一本より、**階層的な順位づけ**を勧めます。

## 主順位

1. `Orbit(d_z)` または `AUC_orbit`
2. `m_orbit`
3. `Obs(d_z)` または `AUC_obs`
4. `m_obs`
5. `Recon(d_z)`

## 補助順位

* `Topo` の維持
* `Trust(2)` など chart の自然さ
* ノイズ頑健性
* 冗長度

こうすると、「どの run が何に強いか」が残ります。
VAE のように

* orbit は良い
* chart はやや良い
* reconstruction は悪い

という run も、潰さずに読めます。

# 合成データセットではさらに強くなる

`PhaseTorus12` のような制御系では、この基盤に

* 因子 readout
* 因子別崩壊順序
* ノイズ帯域ごとの劣化率

をそのまま載せられます。

たとえば

* `m_phase`: `θ1, θ2` の readout が閾値を超える最小 `m`
* `m_noise_reject`: ノイズ因子が十分に捨てられる最小 `m`
* `noise_slope_low/high/band`: ノイズ強度に対する `Topo` や `Orbit` の劣化率

のような量が作れます。

これで Lattice と合成系を、かなり同じ見取り図で読めます。

# 私のおすすめする一番簡潔な定式化

この基盤を一言で言うと、

> **各 run を、full latent の点ではなく、truncation profile の曲線として評価する**

です。

これができると、

* latent8 が本当に良いのか
* latent4 で十分なのか
* 2D chart はどこで成立するか
* 位相構造はどこで死ぬか

が一つの枠組みで見えます。

# いまの Lattice 表に対する最小修正案

大きく作り替えなくても、まずは次の追加だけでかなり前に進めます。

各 run について

* PCA EVR はすでにあるので、その PCA 基底を使う
* `m=1,2,...,min(d_z,4)` くらいまで truncated latent を作る
* 各 `m` で

  * `MSE`
  * `SL₂(Z) mean dist`
  * `max Spearman vs log10_abs_j`
  * `trust`
  * `PH(H1)` の簡易量
    を出す

これだけで、`latent4` と `latent8` の比較がかなり統一されます。
